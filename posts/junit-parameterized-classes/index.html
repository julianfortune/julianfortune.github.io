Nested Parameterization in Junit (5.13+)

2025-07-20
kotlin , jvm , testing

Junit Parameterization

For my day job I write a lot of Junit tests in Kotlin, and up until the recent release of Junit 5.13 it had some rough edges with parameterization: (1) parameterizing multiple methods in a test suite required redundant boilerplate, and (2) achieving cartesian products of parameters was cumbersome and inflexible.

For instance, if we needed to test a greet() function that takes a language and person’s name:

1fun greet(language: Language, name: String): String

and a very similar bidFarewell() function:

1fun bidFarewell(language: Language, name: String): String

in order to thoroughly test these methods we would end up writing something like this:

 1class ConversationTests {
 2
 3  @ParameterizedTest
 4  @CsvSource(value = [
 5    "English | Harry | Hello, Harry",
 6    "Shakespearean | Harry | Hello, Harry",
 7    "PirateSpeak | Harry | Hello, Harry",
 8  ])
 9  fun greet(language: Language, name: String, expected: String) {
10    val result = greet(language, name)
11    assertThat(result).isEqualTo(expected)
12  }
13
14  @ParameterizedTest
15  @CsvSource()
16  fun bidFarewell(language: Language, name: String, expected: String) {
17    val result = bidFarewell(language, name)
18    assertThat(result).isEqualTo(expected)
19  }
20
21}

Language.English Language.Shakespearean Language.PirateSpeak

So I’m realizing this feature is ideal when you have multiple different dependencies / strategies you want to subject a class to while exercising the various methods..

ParameterizedClass Annotation

The new annotation introduced, @ParameterizedClass, can be applied to classes (and nested classes).

…

 1@ParameterizedClass
 2@CsvSource(value = ["1", "2", "3"])
 3class FooBarTests {
 4
 5  @Parameter
 6  lateinit var parameter: Int
 7
 8  @Test
 9  fun something() {
10    // ...
11  }
12}
